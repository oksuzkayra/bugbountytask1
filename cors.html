Cross-Origin Resource Sharing Nedir?

Kökenler arası kaynak paylaşımı anlamına gelir. Ek HTTP başlıkları kullanılarak bir origin'de çalışan web uygulamasının, farklı origin'de yer alan web uygulamasına erişim izni kontrolünü sağlayan HTTP Header tabanlı mekanizmadır. Web uygulaması, tarayıcı üzerinden farklı bir origin'e herhangi bir istek gönderirse cross-origin HTTP isteği oluşturmuş olur.

Sonuç olarak Same-Origin Policy ile zıt çalışır diyebiliriz. SOP erişimi engeller, CORS ise izin verir. Aşağıda örnek olarak verilen istek türleri için CORS ayarlaması yapılabilir.

  • Ajax çağrıları adını verdiğimiz XMLHttpRequest ve fetch() API çağrıları
  • Web fontlarının kullanımı (CSS üzerinden @font-face ile yapılan çağrımlar)

Örnek olarak kayra.org üzerinden example.com'a bir istek gönderirsek, tarayıcı tarafından Origin header'ına kayra.org URL'inin otomatik atandığını görebiliriz.

GET /posts HTTP/1.1
Host: https://example.com
Origin: https://www.kayra.org
...

CORS ayarlaması yapılmış sunucu Origin header'ını kontrol eder. Eğer ilgili origin'in izni var ise Access-Control-Allow-Origin header'ına ilgili origin'i atar ve sonuç olarak aşağıdaki gibi bir response gönderir.

HTTP/1.1 200 OK
Access-Control-Allow-Origin: https://www.kayra.org
...
